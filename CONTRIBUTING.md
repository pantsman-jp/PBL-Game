# CONTRIBUTING.md

## 開発フロー
- ブランチを切って作業
- PR を出し、レビュー後にマージ

## ブランチルール
参考 Issues [#42](https://github.com/pantsman-jp/PBL-Game/issues/42)

ブランチ名は、カテゴリを `/` で区切り、単語を `-` でつないだ簡潔な英語にする
- 機能追加のブランチ名は `feature/追加機能名` とする
  - ex) `feature/quiz-system`
- バグ修正のブランチ名は `bugfix/修正内容` とする
  - ex) `bugfix/player-collision`
- リファクタリングのブランチ名は `refactor/改善内容` とする
  - ex) `refactor/remove-unused-code`

## コミットメッセージ
- 作業内容を簡潔に示す
- 日本語でも英語でもいいです

## PR 作成手順
参考 Issues [#40](https://github.com/pantsman-jp/PBL-Game/issues/40)
- タイトルは日本語で明確に
- テンプレートに従って書いてください
